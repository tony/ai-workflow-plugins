# Lean rubric

The shared reference for the `lean-writing` skill and `/lean:tighten`.
Curated judgment, not an exhaustive regex registry — for heavy,
signature-driven detection with commits, use `/slop:scan`.

## Slop signatures to flag

- **AI signatures** — "Generated by", "Certainly!", "Here is…",
  unexplained emojis, tool metadata.
- **Brittle references** — hard-coded line numbers, file or test
  counts, dated "as of" claims, bare SHAs, local absolute paths (unless
  they are evidentiary, such as a benchmark log).
- **Diff narration** — restating what moved, was renamed, or removed;
  the diff and commit message already carry it.
- **Branch-internal narrative** — intermediate states, abandoned
  approaches, or "no longer" behavior a released user never saw.
- **Prose inflation** — comprehensive, robust, seamless,
  production-ready, leverage, delve, tapestry, best practices.
- **Coded labels** — `[R1]`, `A1`, `Option B` in human-facing text.

## Preserve — do not strip

- Links, citations, anchors, and warnings that still substantiate a
  claim.
- Comments documenting an invariant, protocol constraint, platform
  quirk, security boundary, or upstream workaround.
- Exact counts, dates, and SHAs used as evidence (benchmarks, release
  notes, stack traces, lockfiles).

When unsure, leave the text in place and ask.

## Don't slop while deslopping

Replacements are concrete and shorter than the original. Never narrate
your own edit inside the artifact.

## Contested calls

### Add a table?

- **Use when** — the data is genuinely matrix-shaped, stable, and read
  by cross-column comparison.
- **Avoid when** — you can say it in prose or a short bullet list, the
  cells are long, or the rows churn often.
- **Tradeoff** — tables render cleanly but diff badly and wrap poorly;
  nested "Use when / Avoid when / Tradeoff" headings usually read
  better and edit easier.

### Add a file?

- **Use when** — it draws a durable boundary: a distinct
  responsibility, independent reuse by real callers, or a split of an
  oversized, high-touch module.
- **Avoid when** — it is a single-use helper, a one-line re-export, or
  a speculative future component.
- **Tradeoff** — a new file adds a navigation hop; earn it with a
  boundary, not with line count.

### Add a test?

- **Use when** — it pins a reproduced regression, a public behavior, or
  a distinct failure mode.
- **Avoid when** — it asserts a constant, re-checks an outcome another
  layer already covers, or exists only to raise coverage.
- **Tradeoff** — extend the nearest coherent test before adding a file;
  test through the public path, not implementation details.
